Attendance QR
This site is fully static
HTML, CSS and JavaScript only — no build step, hosted for free on GitHub Pages.

Everything you're using right now runs in Demo Mode: a sample organization, a simulated Google Calendar, real QR codes, and a live dashboard — all stored locally in your browser, with no Google account involved. That matches how the demo environment was scoped: safe to click around in, one button to reset.

Turning on Production Mode
Real Google Calendar + Sheets sync

Production Mode needs a Google Cloud project (Calendar API + Sheets API enabled, an OAuth Client ID pointed at your GitHub Pages URL) — that part can run straight from this static site using Google Identity Services in the browser, no server required.

The one piece that genuinely needs a server: automatically detecting a new calendar event the instant it's created. Google's push notifications for Calendar have to call a real server endpoint, which GitHub Pages can't provide. The workaround is either polling the Calendar API while the dashboard tab is open, or adding one small serverless function (Cloudflare Workers, Vercel, or a Supabase Edge Function all have free tiers) that Calendar can notify.

Everything else — reading the master sheet, writing attendance back to Sheets, generating QR tokens — is the same logic already running in Demo Mode, just pointed at real Google APIs instead of local storage.

Deploying this repo

Push this folder to a GitHub repository, then in Settings → Pages set the source to your main branch, root folder. GitHub gives you a live URL within a minute. Full steps are in README.md at the root of the project.
